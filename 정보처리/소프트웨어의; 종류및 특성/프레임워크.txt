    프레임워크
정의 : 목적 달성을 위해 복잡하게 연결 문제해결 구조이자 소프트웨어 개발의 뼈대

    소프트웨어 프레임워크
-응용 프로그램 개발, 문제해결 하기위한 설계와 구현을 재사용 가능한 형태인 소프트웨어 환경
-코드 라이브러리, 애플리케이션 인터페이스, 설정 정보 등의 집합
-다양한 소프트웨어 개발을 가능한 여러 형태의 컴포넌트
-완제품이 아니며 개발자의 코드 유저 데이터가 필요

    특징
모듈화(Modularity) : 단일 기능을 갖출 수 있도록 부분적으로 묶어서 처리하는 기술
재사용성(Reusabillity) : 전체나 일부 소프트웨어가 더룬 응용 목적으로 사용되는 정도
확장성(Extensibillity) : 추가적인 설계 없이 확장이 얼마나 쉽고 가능한가의 용이성
제어의 역 흐름(Inversion of Control) : 프레임워크 코드가 전체 앱 흐름의 제어 하여
특정 이벤트가 발생 할때 다형성을 통해 앱이 확장한 메소드를 호출하여 제어가 소프트웨어
프레임워크뷰터 앱으로 꺼꾸로 흐르게 한다

    기대 효과
개발 용이성 : 필요한 기능을 미리 제공받아 빠른시간 개발 가능
품질 보증 : 오류 발생 최소화 품질 향상
변경 용이성 : 기능을 추가/수정을 부담 없이 할 수 있다
유지보수 용이성 : 체계적이고 안정적이며 변경되더라도 위험부담을 줄일수있다
재사용성 향상 : 컴포턴트들을 타 사업에서 사용할수 있어 재사용성이 극대화 가능
표준화율 향상 : 전형화된 개발 기술, 방법을 제공하여 설계, 새발 등 시스템 구축에 표준 제시
상호 운용성 향상 : 전자 정부 표준 프레임워크 기반으로 개발된 시스템간에
컴포넌트 연계가 용이하여 상호 운용성ㅇ이 극대화 가능
