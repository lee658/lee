    미들웨어(Middleware)

    미들웨어 정의
-운영체제와 소프트웨어 앱 사이에 위치하는 미들웨어는 소프트웨어 앱에게
제공하는 서비스를 추가/확장 하여 제공하는 컴퓨터 소프트웨어
-클라이언트와 서버 간통신을 담담하는 시스템 소프트웨어 

    미들웨어의 종류
DBMS(DataBase Management System)
-데이터베이스 벤터에서 제공하는 클라이언트에서 데이터베이스와 연결하기위한 미들웨어
-이제품을 사용해 시스템 구축경우 2티어 아키텍처라고 한다

RPC(Remote Procedure Call)
-앱의 프로시저를 사용하여 원격 프로시저를 마치 로컬 프로시저 처럼 호출하는 미드웨어

MOM(Message Oriented Middleware)
-메시지 기반의 비동기형 메세지를 전달하는 방식의 미드뤠어로 
이 기종 분산 데이터 시스템의 데이터 동기를 위해 많이 사용한다

TP-Monitor
-트랜잭션이 올바르게 처리되는지 감시/제어 프로그램  
-온라인 트랜잭션 업무(은행 계정, 항공기/버스 예약 업무)에서 트랜잭션을 처리/감시하는 미들웨이
-사용자 슈가 증가하여도 빠른 응답 속도를 유지해야 하는 업무에 적합

ORB(Obiect Request Broker)
-객체 지향 미들웨어로 코바(CORBA) 표준 스펙을 구현한 미들웨어
-TP-Monitor가 가지는 장점을 추가 구현

WAS(Web Application Sever)
-클라이언트/서버 환경보다는 웹 환경을 구현하기위한 미들웨어
-EJB는 서버 컴퓨터에서 운영되는 Java 컴포넌트들을 설정하는 아키텍처
