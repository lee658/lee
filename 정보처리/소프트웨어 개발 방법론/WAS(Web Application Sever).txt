    WAS(Web Application Sever)
    
    WAS의 개념
-동적인 웹, 앱, 웹 서비스 개발 지원 소프트웨어
-데이터 접근 관리, 세션 관리, 트랜잭션 관리등 위한 라이브러리 제공
-WAS는 HTTP 세션 처리를 위한 웹 서버 기능 과 필수적인 기업 업무가 Java, EJB 컴포넌트 기반으로 구현
-유저가 웹 브라우저로 요청하면 정적 데이터는 웹서버가 직접 처리
-동적 데이터는 웹 서버에서 직접 처리하지 못하여 WAS에서 지원받아 처리
-정적 데이터는 이미지나 자바스크립트 코드 등
-동적 데이터는 데이터베이스와 접속, 외부 시스템과의 연동 등

    WAS의 종류
GlassFlash : GlassFlash Communbity, NetBeans 개발툴과 연동하여 사용

JBoss : Red Hat, JBoss(Java 기반으로 오픈소스 미들웨어의 총칭)제공, 오픈소스 제품 이용

Jetty : Eclipse Foundation에서 제공, 빠르 처리속도가 요구되는 환경에서사용

JEUS : TmaxSoft(국내 미들웨어 시장 1위 기업)제공, 대량의 안정적 처리 요구
적시의 기술 지원이 필요한 경우 오픈소스 사용

Resin : Caucho Technology에서 제공, 빠른처리 속도가 요구되는경우 사용

WebLogic : Oracle Corporation에서제공, 대량의 안정적인 거래처리 요구 사용

WebSphere : IBM에서 제동, 대량의 안정적인 거래처리 요구 사용

    WAS 분석시 고려사항
가용성
-오랜 시간 동안 시스템을 운영할때 장애 발생 가능성 확인
-안정적인 트랜잭션 처리가 가능한지 확인
-패치 설치를 위한 재기동 확인
-WAS 이중화 지원 확인

성능
-대규모 거래 요청 시 처리 성능을 확인
-다양한 설정 옵션을 지원하는지 확인
-GC(Garbage Collection)의 다양한 옵션을 지원하는지 확인

기술지원
-공급 밴더들의 안정적인 기술 지원이 가능한지 확인
-다수의 사용자들 간의 정보를 공유 가능한지 확인
-오픈소스 여부 확인

구축 비용
-라이선스 정책, 비용 확인
-유지/관리 비용 확인
-총 소요비용(TCO) 확인
