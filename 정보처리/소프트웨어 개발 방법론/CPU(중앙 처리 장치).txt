    CPU(중앙 처리 장치)
CISC(Complex Instruction Set Computer)
-복잡하고 많은 종류의 명령어와 주소 지정 모드 사용
-가변 길이 명령어 형식
-100~250개 정도의 많은 명령어를 가지고 있어 설계가 어렵다
-마이크로 프로그래밍(소프트웨어적) 제어 방식이다
-명령어가 소프트웨어적이므로 호환성이 좋다
-명령어를 해석한후 명령어를 실행한다
-컴파일 과정이 쉽고 호환성이 좋지만 속도는 느리다
-인텔(Intel)사의 CPU에 주로 사용

RISC(Reduced Instruction Set Computer)
-간단하고 적은 졸류의 명령어와 적은 수의 주소 지정 모드 사용
-고정 길이 명령어 형식
-CISC에 비해 명령어 수가 적다
-하드웨어(논리 회로를 이용한 하드웨어)적 제어 방식
-효율적인 퍼이프라이닝 구조 사용
-명령어 길이가 미리 정해져 해석 속도가 빠르다
-작고 빠른 명령어 사용을 위해 많은 수의 범용 레지스터가 사용
-처리속도가 빠르고 하드웨어 구조가 간단하다
-효율성이 떨어지고 전력 소모가 적다
-처리 비트가 변하거나 프로세서의 구조가 변하면 하위 프로세서의 호환성이 떨어진다
-고성능의 워크스테이션이나 그래픽용 컴퓨터에서 주로 사용
