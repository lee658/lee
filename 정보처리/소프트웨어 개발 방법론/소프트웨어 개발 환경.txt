    소프트웨어 개발 환경 
운영체제(OS : Operationg System)
-운영체제의 정의 : 하드/소프트웨어 자원관리 컴퓨터 프로그램을 위한 공통 제공 시스템 소프트웨어
    
    운영체제의 종류 
Windos : 중소 규모 서버, 개인용 PC, Tablet PC, Embedded System 등에서 사용

유지/관리 비용 측면에서 Windos 기반 시스템이 강점이 있다

UNIX : 대용량 처리, 안정성이 요구되는 Sever, NAS, Workstation 등에서 사용

신뢰할 수 있눈 대용량 처리를 위해서는 UNIX 기반 시스템 선호

Linux : 증대 규모 서버 등 사용 Linux 기반 시스템이 하드/소프트웨어 비용이 적게 든다

IOS : 애플의 운영체제 스마트폰, 태블릿 PC에 사용

Android : 구글의 개방형 운영체제 스마트폰, 태블릿PC에 사용

    운영체제 분석시 고려사항 
신뢰도 : 오랜 시간 시스템을 운영할 때 운영체제 고유의 장애 발생 가능성 확인

성능 : 다수의 동시 사용자 요청 처리가 가능 확인

대량 파일 작업 처리 가능 확인

기술지원 : 공급 벤더(Vender)들의 안정적인 기술 지원이 있는지 확인

주변 기기 : 다수의 주변 기기 지원 여부 확인

구축 비용 : 지원 가능한 하드웨어 비용, 설치할 앱의 라이선스 정책, 비용 확인(TCO 확인)

    운영체제 현황
-32bit 운영체제는 4gb메모리 까지 엑세스 가능(사용자 2gb)
-64big 운영체제는 4gb 이상의 메모리까지 엑세스 가능
-CISC 설계 방식이 적용된 인텔의 x86 아키텍처 기반 칩을 사용하고 있는
하드웨어는 Windos, Linux를 운영체제로 설치할 수 있다
-RISC 설계방식이 적용 칩들은 UNIX 운영체제를 설치한다
-HP와 Intel 사가 협력해서 만든(IA -64)칩은 여러 운영체제 지원
-RISC 설계방식이 적용된 ARM칩은 스마트폰 태블릿에 주로 채택되고
Ios, Android 등의 운영체제를 지원한다
